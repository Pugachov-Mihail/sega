# Архитектура эмулятора

Проект моделирует работу 16/32-битного процессора.

## 🏗 Структура CPU (`src/cpu.rs`)

Центральная часть проекта — структура `CPU`, которая содержит состояние «железа»:

### 1. Регистры
- **`d: [u32; 8]`**: Регистры данных (D0-D7). Предназначены для арифметических операций.
- **`a: [u32; 8]`**: Адресные регистры (A0-A7). Используются для хранения указателей на память.
  - *Важно:* В текущей реализации **A4** (индекс `a[4]`) используется как указатель стека (**Stack Pointer**).
- **`pc: u32`**: Program Counter (Счетчик команд). Указывает на текущий адрес выполняемой инструкции.
- **`sr: u16`**: Status Register (Регистр состояния). Хранит флаги результатов операций:
  - **Z (Zero)** — бит 2: результат ноль.
  - **N (Negative)** — бит 3: результат отрицательный (старший бит установлен).
  - **C (Carry)** — бит 0: перенос при сложении.

### 2. Память
- Эмулятор выделяет 4 МБ памяти (`vec![0; 4 * 1024 * 1024]`).
- Память инициализируется нулями.
- Используется порядок байтов **Big-Endian** (старший байт по младшему адресу).

### 3. Стек
Реализован классический стек (Full Descending):
- Указатель стека указывает на последнюю занятую ячейку.
- **PUSH32:** Уменьшает SP на 4, затем записывает значение.
- **POP32:** Читает значение по SP, затем увеличивает SP на 4.

## 🔄 Жизненный цикл команды
Процессор работает в три этапа:
1. **Fetch (Выборка):** Считывание 2-байтового опкода из памяти по адресу `pc`, увеличение `pc` на 2.
2. **Decode (Декодирование):** Сопоставление байтов с перечислением `Instruction` (бит-хакинг и маскирование).
3. **Execute (Выполнение):** Изменение состояния регистров, памяти или `pc` на основе команды.
